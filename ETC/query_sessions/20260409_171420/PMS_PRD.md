## 1. 개요 및 목표

### 1.1 프로젝트 배경

IT 회사 내에서 분산되어 관리되던 프로젝트 정보를 통합 관리하고, 일정 조율 및 모니터링의 효율성을 높이기 위한 통합 프로젝트 관리 시스템 구축이 필요함.

### 1.2 프로젝트 목적

- 프로젝트 정보의 중앙 집중화 및 표준화된 관리
- 실시간 일정 시각화를 통한 효율적인 일정 관리
- 역할 기반 접근 제어를 통한 보안 강화
- 웹 및 모바일 플랫폼을 통한 접근성 확보

### 1.3 기대 효과

- 프로젝트 일정 중첩으로 인한 리소스 충돌 감소
- 관리자 및 임원의 실시간 프로젝트 현황 모니터링
- 개발자의 프로젝트 정보 입력 및 수정 업무 효율화
- 모바일 환경에서의 원격 작업 지원

## 2. 사용자 스토리

### 2.1 사용자 페르소나

#### PM (프로젝트 매니저)
- **역할**: 프로젝트 진행 관리 및 팀원 coordination
- **목표**: 프로젝트 일정을 효율적으로 관리하고 팀원들과 공유
- **사용 빈도**: 일상적 (하루 2-3 회)

#### 관리자 (PMO/임원)
- **역할**: 전사 프로젝트 모니터링 및 의사결정 지원
- **목표**: 다수 프로젝트의 진행 상황 파악 및 리소스 배분 최적화
- **사용 빈도**: 주간 (주 2-3 회)

#### 개발자/설계자
- **역할**: 프로젝트 수행 및 일정 보고
- **목표**: 자신의 업무 일정과 프로젝트 일정 관리
- **사용 빈도**: 프로젝트 진행 시 (필요시)

## 3. 기능적 요구사항 상세

### 3.1 프로젝트 정보 관리 기능

#### FR-01: 프로젝트 생성
- **설명**: PM 이新项目 프로젝트를 생성하고 기본 정보를 입력
- **입력 항목**:
  - 프로젝트명 (필수, 최대 100 자)
  - 프로젝트코드 (필수, 고유값, 영문/숫자/하이픈)
  - 고객사 (필수)
  - 설계자 (선택, 다중 선택)
  - 개발자 (선택, 다중 선택)
  - 시작일 (필수)
  - 종료일 (필수, 시작일 이후)
- **제약 조건**:
  - 프로젝트코드는 중복 불가
  - 종료일은 시작일보다 앞서면 안 됨

#### FR-02: 프로젝트 조회
- **설명**: 사용자 역할에 따라 조회 가능한 프로젝트 범위 제한
- **PM**: 자신의 프로젝트 전체 조회
- **관리자**: 모든 프로젝트 조회
- **개발자/설계자**: 참여 프로젝트 조회
- **정렬**: 시작일, 종료일, 고객사, 프로젝트명 기준 정렬 가능
- **필터**: 기간, 고객사, 진행상태 필터

#### FR-03: 프로젝트 수정
- **설명**: 프로젝트 정보 수정
- **권한**: 프로젝트 생성자 또는 관리자만 수정 가능
- **수정 항목**: 프로젝트명, 고객사, 설계자, 개발자, 일정
- **기록**: 수정 이력 로그 유지

#### FR-04: 프로젝트 삭제
- **설명**: 프로젝트 삭제 (소프트 딜리트)
- **권한**: 관리자만 삭제 가능
- **처리**: 삭제 플래그만 변경, 실제 데이터는 보관

### 3.2 일정 관리 기능

#### FR-05: 간트 차트 시각화
- **설명**: 프로젝트 일정을 시각적 간트 차트로 표시
- **기능**:
  - 프로젝트별 간트 바 표시
  - 기간별 (일/주/월) 뷰 전환
  - 드래그 앤 드롭으로 일정 조정
  - 확대/축소 기능
  - 프로젝트 간 중첩 시각화
- **출력**: 브라우저 기반 렌더링 (서버 side rendering 아님)

#### FR-06: 일정 중복 감지
- **설명**: 동일 리소스 중복 할당 감지
- **감지 항목**:
  - 동일 설계자/개발자가 중복된 프로젝트 일정
  - 프로젝트 일정 기간 내 자원 충돌
- **경고**:
  - 실시간 검증 (일정 입력 시)
  - 경고 메시지 표시
  - 조정 제안 기능

#### FR-07: 일정 조정 및 모니터링
- **설명**: 프로젝트 일정 변경 관리
- **기능**:
  - 시작일/종료일 변경
  - 변경 이력 추적
  - 일정에 따른 의존성 표시
  - 마일스톤 관리

### 3.3 권한 관리 기능

#### FR-08: 역할 기반 접근 제어 (RBAC)
- **역할 정의**:
  - **Admin**: 전체 권한 (프로젝트 CRUD, 사용자 관리, 시스템 설정)
  - **PM**: 자기 프로젝트 CRUD, 프로젝트 조회 (전체), 일정 조정
  - **Developer**: 참여 프로젝트 조회, 일정 확인, 내 정보 수정
  - **Designer**: 참여 프로젝트 조회, 일정 확인, 내 정보 수정
  - **Guest**: 프로젝트 조회만 가능 (관리자 권한으로 제한)
- **권한 정책**:
  - 역할별로 접근 가능한 API 엔드포인트 제한
  - API 레벨에서 권한 검증

#### FR-09: 사용자 관리
- **설명**: 관리자용 사용자 CRUD
- **기능**:
  - 사용자 생성 및 등록
  - 역할 변경
  - 계정 활성화/비활성화
  - 비밀번호 초기화
- **입력**: 이름, 이메일, 역할, 부서

#### FR-10: 인증 및 세션 관리
- **설명**: 안전한 사용자 인증
- **기능**:
  - 이메일/비밀번호 기반 인증
  - JWT 토큰 기반 세션 관리
  - 세션 만료 및 자동 로그아웃
  - 비밀번호 암호화 저장 (bcrypt)

### 3.4 플랫폼 기능

#### FR-11: 반응형 웹 UI
- **설명**: 다양한 화면 크기에 자동 대응
- **지원 화면**:
  - 데스크탑 (1920x1080 이상)
  - 태블릿 (768px-1024px)
  - 모바일 (320px-767px)
- **특징**:
  - 터치 인터페이스 지원
  - 모바일 네이티브 느낌의 UX

#### FR-12: 모바일 최적화
- **설명**: 모바일 환경 전용 최적화
- **기능**:
  - PWA (Progressive Web App) 지원
  - 오프라인 캐싱
  - 푸시 알림 (일정 변경, 중복 감지)
  - 터치 최적화된 인터랙션

### 3.5 시스템 관리 기능

#### FR-13: 로그 관리
- **설명**: 사용자 활동 및 시스템 로그 기록
- **로그 항목**:
  - 로그인/로그아웃
  - CRUD 작업
  - 권한 변경
  - 오류 발생
- **유지기간**: 6 개월

#### FR-14: 데이터 백업
- **설명**: 정기적 데이터 백업
- **빈도**: 일일 백업, 주간 아카이브
- **복구**: 24 시간 내 복구 가능 목표

## 4. 비기능적 요구사항

### 4.1 성능 요구사항

| 항목 | 목표치 | 측정 방법 |
|------|--------|----------|
| 페이지 로딩 시간 | 2 초 이내 | Lighthouse 점수 |
| API 응답 시간 | 200ms 평균, 500ms 최대 | 모니터링 도구 |
| 동시 사용자 | 100 명 이상 | 부하 테스트 |
| 데이터 조회 | 1000 개 프로젝트 이내 1 초 | 실제 데이터 기반 |
| 간트 차트 렌더링 | 500ms 이내 | 차트 크기 100个项目 기준 |

### 4.2 보안 요구사항

| 항목 | 요구사항 |
|------|----------|
| 인증 | JWT 토큰 기반, 30 분 세션 만료 |
| 비밀번호 | bcrypt 해시, 최소 8 자, 대소문자/숫자/특수문자 |
| 권한 | 역할 기반 접근 제어, API 레벨 검증 |
| 데이터 | HTTPS 필수, SQL injection 방지, XSS 방지 |
| 로깅 | 모든 인증 시도 로그, 이상 행위 모니터링 |
| 백업 | 일일 자동 백업, 3 개월 보관 |

### 4.3 신뢰성 요구사항

| 항목 | 목표치 |
|------|--------|
| 가용성 | 99.5% (월간 다운타임 3.6 시간 이내) |
| 데이터 무결성 | ACID 준수, 트랜잭션 보장 |
| 오류 처리 | Graceful degradation, 사용자 친화적 메시지 |
| 복구 시간 목표 (RTO) | 4 시간 이내 |
| 복구 지점 목표 (RPO) | 1 시간 이내 |

### 4.4 usability 요구사항

| 항목 | 목표치 |
|------|--------|
| 학습 곡선 | 30 분 내 기본 기능 숙지 |
| 사용자 만족도 | NPS 70 이상 |
| 접근성 | WCAG 2.1 AA 준수 |
| 모바일 사용률 | 전체 사용의 30% 이상 |

### 4.5 호환성 요구사항

| 플랫폼 | 버전 요구사항 |
|--------|--------------|
| Chrome | 최신 2 개 버전 |
| Firefox | 최신 2 개 버전 |
| Safari | 최신 2 개 버전 |
| Edge | 최신 2 개 버전 |
| 모바일 iOS | iOS 14 이상 |

### 4.6 확장성 요구사항

- **사용자 수**: 1,000 명 이상으로 확장 가능
- **프로젝트 수**: 월 500 개 이상 추가 가능

## 5. 기술 스택 제안

### 5.1 백엔드 스택

| 기술 | 역할 | 선택 이유 |
|------|------|----------|
| Python 3.11+ | 프로그래밍 언어 | 명확한 문법, 풍부한 라이브러리, 타입 힌트 지원 |
